voice_query

command: load_dataset

voice_query: problem type

Loads nothing extra — run load titanic first. Routes to problem_type and returns a speech frame.

voice_query: preprocessor

Requires ENABLE_PREPROCESSOR=1 on the server. Run load sample first. Returns speech + updated panels on the versioned copy (v0 preserved).

voice_query: trainer

Requires ENABLE_TRAINER=1 and cleaned data. Streams real per-epoch training_update frames (train/val loss) then speech.

voice_query: evals

Requires ENABLE_EVALS=1 and cleaned data (load sample + clean data first). Returns speech + confusion-matrix panels from a real sklearn test split.

command: load_run

command: narrate

Reads accumulated EDA findings + active-run verdict from Redis, no recomputation. Returns a report frame rendered below.

command: reset

tips

EDA queries → router picks eda
"what problem are we solving?" → problem_type
ENABLE_PREPROCESSOR=1: clean data / ready to train → preprocessor
"is my model overfitting?" → training_monitor
Click load_run first if you want a non-default run
Click load_dataset if you want sample.csv instead of titanic
🎙 mic: click once to start, click again to stop. The recording is posted to /transcribe (Whisper) and the returned text is sent as a voice_query automatically.
